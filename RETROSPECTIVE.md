# Project Retrospective

## Introduction

This project retrospective reflects on the development process of my photography portfolio website, “Memories Captured by Donna Baca.” Throughout this project, I gained experience in project management, website development, GitHub organization, HTML, CSS, troubleshooting, and creative design. This reflection is based on lessons learned, project communication, risk management, planning, and continuous improvement.

---

# What Went Well

One aspect that went well during this project was the overall visual design and professionalism of the final website. I successfully created a multi-page photography portfolio that reflected the aesthetic and branding vision I originally planned. The final project includes organized navigation, external CSS styling, embedded multimedia, image galleries, and a clean layout that feels cohesive and professional.

Another success was learning how to manage and organize files using GitHub and GitHub Pages. At the beginning of the project, I had very little experience using GitHub. By the end of the project, I learned how to organize repositories, create folders for images and videos, manage file paths, and publish a live website online.

Peer feedback also played an important role in improving the project. Based on suggestions from classmates, I revised the color palette, hover effects, spacing, and gallery organization. PMUPM Chapter 7 discusses the importance of communication and continuous improvement throughout a project lifecycle, and peer feedback helped strengthen both the design and usability of my website.

Additionally, breaking the project into smaller tasks helped improve workflow management. Completing development in phases made the project feel more manageable and reduced stress during the final stages of submission.

---

# What Went Wrong

One major challenge throughout this project was technical troubleshooting. Since I am still a beginner in coding and GitHub, small mistakes such as missing div tags, incorrect file paths, broken media links, or CSS formatting issues often caused pages not to function correctly. Some issues took several hours to identify and resolve.

Another challenge involved time management. Certain tasks required much more time than originally expected, especially troubleshooting GitHub Pages deployment errors and organizing multimedia content. PMUPM Chapter 7 explains how risks and unexpected obstacles can affect project timelines, and this project demonstrated how technical risks can significantly slow development progress.

Communication and documentation also became important lessons during this project. At times, I became overwhelmed by multiple revisions and troubleshooting steps. Looking back, documenting changes more carefully throughout the process would have made debugging easier later.

Finally, learning GitHub while simultaneously learning web development created a steep learning curve. Although the project was rewarding, the technical complexity was more difficult than I originally anticipated.

---

# What I Would Do Differently

If I were to complete this project again, I would spend more time planning the repository structure before beginning development. Organizing folders, naming files consistently, and planning page layouts ahead of time would have prevented several technical issues later in the project.

I would also create wireframes and a clearer development schedule before coding. PMUPM Chapter 7 emphasizes the value of lessons learned and process improvement, and I realized that stronger planning at the beginning of a project can reduce confusion and rework later.

Another improvement would be practicing GitHub and HTML skills earlier in the semester. Becoming more comfortable with repository management and troubleshooting beforehand would have reduced stress during the final stages of development.

Lastly, I would document project updates and coding changes more consistently throughout the process. Keeping track of revisions, fixes, and testing results would improve efficiency and make future troubleshooting easier.

Overall, despite the challenges, this project helped me build confidence in coding, project management, creative problem solving, and web design. The experience demonstrated the importance of planning, adaptability, communication, and continuous improvement throughout a project lifecycle.
